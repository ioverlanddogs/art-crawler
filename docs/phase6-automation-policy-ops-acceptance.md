# Phase 6 Acceptance Criteria

Phase 6 is complete when all are true:

## Automation controls
- operator can see which rules are disabled, shadow, or active
- rule scope and status are understandable
- recent automated actions are visible
- rollout states are explicit and textual

## Bulk workflows
- bulk actions preview scope before execution
- destructive bulk actions require stronger confirmation
- duplicate/conflict risks are explained
- reason capture is supported where appropriate
- unsupported undo/rollback is stated honestly

## Escalation
- exception queue is visible
- operator can see why work was escalated
- human override path is clear
- return-to-automation state is visible where supported

## Explainability
- UI explains why automation matched or escalated
- confidence/threshold context is visible
- human-review boundaries are explicit
- overrides remain auditable

## UX
- no critical state relies on color alone
- automation states are readable and scanable
- partial telemetry degrades gracefully
- focus and keyboard behavior remain accessible
